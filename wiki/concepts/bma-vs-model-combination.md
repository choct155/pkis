## Definition
**Bayesian model averaging (BMA):** $p(\mathbf{X}) = \sum_{h=1}^{H} p(\mathbf{X}|h)\,p(h)$. A single model $h$ is assumed to have generated the *entire* dataset; the sum over $h$ reflects prior uncertainty about which model that is, and this uncertainty collapses to a point as $N\to\infty$.

**Model combination / mixture models:** $p(\mathbf{X}) = \prod_{n=1}^{N}\sum_{z_n} p(x_n, z_n)$. Different data points can be generated by different components; the latent variable $z_n$ is per-observation.

### Why it matters
The distinction is subtle but operationally critical: BMA is not an ensemble — in the large-data limit it concentrates on one model. Model combination, by contrast, is a richer generative model that persists as a genuine mixture regardless of sample size. Confusing the two leads to incorrect inference; e.g., treating a mixture of Gaussians as BMA would wrongly imply the mixture should collapse to a single Gaussian as data accumulate.
