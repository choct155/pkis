The Confusion Matrix — What the LLM Actually Does

The LLM renders a binary judgment on each extraction: correct (L=1) or incorrect (L=0). The true accuracy A is the latent variable we want to estimate. The confusion matrix records how often L agrees with A across all four possible outcomes.

Key observation The LLM's overall accuracy of 75% is a marginal — it conflates sensitivity and specificity weighted by true prevalence p. Two different confusion matrices can produce the same 75% overall accuracy with very different error structures. You cannot recover p from the silver sample alone.
Stage 2 of 3

Prevalence Estimation — Debiasing with the Gold Sample

The gold sample gives you empirical estimates of sensitivity and specificity. With those in hand, you can invert the confusion matrix to recover the true prevalence p̂ — the fraction of extractions that are actually correct.

The prevalence estimation formula — derivation
// Observe: P(L=1) = P(L=1|A=1)·P(A=1) + P(L=1|A=0)·P(A=0)
// Substitute: P(A=0) = 1 − p, P(L=1|A=0) = 1 − specificity
P̂(L=1) = sensitivity · p + (1 − specificity) · (1 − p)
// Rearrange for p:
p̂ = ( P̂(L=1) − (1 − specificity) ) / ( sensitivity − (1 − specificity) )
// Equivalently:
p̂ = ( P̂(L=1) − FPR ) / ( TPR − FPR )
// This is the standard roc-based prevalence estimator

The identification problem This formula requires knowing the true sensitivity and specificity — which come from the gold sample. If the gold sample is too small, the confusion matrix estimates are noisy, and the prevalence estimate inherits that noise. This is what we fix in Stage 3.
Stage 3 of 3

Bayesian Updating — How Each Gold Observation Shifts Your Uncertainty

Before seeing any gold observations, you have prior uncertainty over sensitivity and specificity. Each gold observation — a human judgment H on a case where you also have L — updates your posterior beliefs about those parameters. Watch the distributions sharpen as you add observations.

We model sensitivity ~ Beta(α_s, β_s) and specificity ~ Beta(α_q, β_q). The Beta distribution is the natural conjugate prior for a Bernoulli success probability. Each concordant or discordant observation updates the relevant Beta parameters.

Bayesian update rules — Beta conjugate model
// Prior: sensitivity ~ Beta(α_s, β_s), specificity ~ Beta(α_q, β_q)
// Observation types and their updates:
TP: H=1, L=1  →  α_s += 1  (confirms LLM caught a correct extraction)
FN: H=1, L=0  →  β_s += 1  (LLM missed a correct extraction → sensitivity failure)
TN: H=0, L=0  →  α_q += 1  (confirms LLM caught a wrong extraction)
FP: H=0, L=1  →  β_q += 1  (LLM passed a wrong extraction → specificity failure)

// Posterior mean: E[sensitivity] = α_s / (α_s + β_s)
// Posterior variance: Var[sensitivity] = α_s·β_s / [(α_s+β_s)²·(α_s+β_s+1)]
// 95% credible interval via Beta quantiles

The discordant observation insight FP and FN observations are disproportionately informative. They directly constrain the error cells of the confusion matrix — the cells you know least about. A gold sample that happens to contain only TP and TN observations gives you almost no information about the failure modes, even if N_gold is large. Stratified sampling that deliberately seeks discordant cases is more efficient than random sampling.
