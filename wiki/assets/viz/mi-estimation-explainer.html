Histogram & Binning Estimator

Family 1 of 4 · Density estimation via grid discretization

Mutual information measures how much knowing one variable reduces uncertainty about another. The histogram estimator is the most direct translation of the definition into a computation: discretize both variables into bins, count co-occurrences, treat counts as probabilities, plug into the MI formula.

It is the right estimator to understand first because its failure modes are completely transparent — every limitation is visible in the formula itself. All subsequent families are motivated by trying to fix the specific things that go wrong here.

The definition we are estimating

Mutual information — definition
I(X ; Y) = Σx,y P(x,y) · log [ P(x,y) / ( P(x) · P(y) ) ]

// P(x,y) = joint distribution over both variables
// P(x), P(y) = marginal distributions — what you'd get ignoring the other variable
// The log ratio measures how much the joint differs from independence
// When X ⊥ Y: P(x,y) = P(x)P(y), log ratio = 0, I = 0
// When X = Y: I = H(X) — knowing Y tells you everything about X

The histogram estimator replaces the unknown distributions P(x,y), P(x), P(y) with empirical frequency counts from N samples.

What makes this hard We never observe the true distributions — only N samples from them. Every estimate of P(x,y) is a ratio of counts, and ratios of counts have sampling noise. The core challenge is that this noise inflates the apparent MI even when the true MI is zero.

Histogram Mechanics

Step-by-step construction of the estimator
①

Discretize both variables

Partition the range of X into Bx equal-width bins and Y into By bins. This creates a Bx × By grid of cells in the joint space.

Bins: [a0, a1), [a1, a2), ..., [aB-1, aB]
// Equal-width binning is the default but not the only choice
// Equal-frequency (quantile) binning is often better behaved
②

Estimate joint and marginal probabilities

For each cell (i,j) in the grid, count how many samples fall in it. Divide by N to get an estimated probability.

P̂(i,j) = count(X ∈ bin_i AND Y ∈ bin_j) / N
P̂(i) = count(X ∈ bin_i) / N = Σj P̂(i,j)
P̂(j) = count(Y ∈ bin_j) / N = Σi P̂(i,j)
③

Plug into the MI formula

Î(X ; Y) = Σi,j: P̂(i,j)>0 P̂(i,j) · log [ P̂(i,j) / ( P̂(i) · P̂(j) ) ]
// Only sum over non-empty cells (log 0 is undefined)
// Use nats (log base e) or bits (log base 2) consistently
④

The bias — why the estimator is not the truth

The expected value of Î is not I. Even for independent X and Y (true MI = 0), the estimator returns a positive value. The analytical bias is approximately:

𝔼[Î] ≈ I + (Bx · By − Bx − By) / (2N)
// The red term is the bias — grows with bin count, shrinks with N
// With 10×10 bins and N=500: bias ≈ (100-10-10)/(1000) = 0.08 nats
// For comparison: I(X;Y) = 0.5 nats for Gaussian with ρ=0.7
// Bias is 16% of the true MI — not negligible

The bias arises because count ratios are not unbiased probability estimates. Even if the true P(i,j) = P(i)·P(j) (independence), the ratio of noisy counts will deviate from 1, and its log will be non-zero on average.

Miller-Madow correction A common bias correction subtracts the estimated bias term: ÎMM = Î − (number of non-empty cells − Bx − By + 1) / (2N). This helps but does not eliminate the bias, especially for small N or high B.

Degradation Cases

When and why the histogram estimator fails

Case 1 — Finite sample bias (always present)

Even with independent X and Y, the estimator returns positive MI. Bias = (B_x · B_y − B_x − B_y) / 2N. With 10×10 bins and N=500 this is 0.08 nats — potentially larger than the true MI you are trying to measure.

Key pointMore bins is not always better. Adding bins without adding samples increases bias faster than it improves resolution.

Case 2 — Bin boundary arbitrariness

The estimated MI depends on where bin boundaries fall. If the true density has mass concentrated near a boundary, small shifts produce large changes in cell counts. Two researchers with different bin placements will get different MI estimates from the same data. This is not a statistical noise problem — it is a structural sensitivity to an arbitrary choice.

Case 3 — Dimensionality collapse

With d-dimensional X, you need B^d cells per variable. With B=2 and d=768 (a typical embedding dimension): 2^768 cells. With any finite sample, essentially all cells are empty. The estimator collapses to 0 or saturates based on the tiny fraction of occupied cells. This is not fixable with more data — it is a structural impossibility.

The curse — why it's catastrophic
Cells needed: Bd
Samples available: N
For meaningful estimation: N >> Bd
// B=3, d=10: need N >> 59,049
// B=2, d=20: need N >> 1,048,576
// B=2, d=768: impossible regardless of N

Case 4 — Discrete-continuous mixing

When one variable is discrete (like edge type τ) and one is continuous (like a query embedding q), the joint distribution has no density in the standard sense — it is a mixture of point masses and continuous densities. The histogram estimator's bin volume calculation assumes a density exists. Bin volume is undefined when the distribution is mixed, so the probability estimate count/(N·bin_volume) is undefined for the discrete dimension.

The practical fix — treating each discrete value as its own bin with unit volume — works but conflates cell sparsity from the continuous dimension with cell counts from the discrete dimension. The bias formula no longer applies cleanly.

When histogram estimation is appropriate Both variables genuinely discrete and low-cardinality (fewer than ~20 values each). Large N relative to total cell count. Need a fast rough estimate, not a precise one. All continuous variables have been pre-discretized with principled bin placement. If these conditions are not met, KSG is a better starting point.

KNN / KSG Estimator

Family 2 of 4 · Density estimation via nearest-neighbor geometry

The histogram estimator fails because it requires explicit density estimation on a fixed grid. KSG (Kraskov, Stögbauer, Grassberger 2004) avoids explicit density estimation entirely — instead of building a grid and counting cells, it uses the geometry of the sample itself to estimate local densities adaptively.

The core insight: the distance from a point to its k-th nearest neighbor encodes local density. In a high-density region, neighbors are close. In a sparse region, they are far. You can express MI directly in terms of these distances without ever fitting a density function.

Why this fixes the histogram problems No bin boundaries — resolution adapts to local data density. Bias is O(1/N) rather than O(B²/N). Works in moderate-to-high dimensions. Has a principled extension to discrete-continuous mixing (Ross 2014).

KSG Mechanics

The digamma derivation — why each term is what it is

Before the formula, we need to understand what the digamma function is and why it appears.

The digamma function

Digamma ψ — definition and key values
ψ(n) = d/dn ln Γ(n) = −γ + Σk=1n-1 1/k    (for positive integers)

ψ(1) = −γ ≈ −0.5772
ψ(2) = 1 − γ ≈ 0.4228
ψ(n+1) = ψ(n) + 1/n    (recurrence relation)

// γ ≈ 0.5772 is the Euler-Mascheroni constant
// ψ(n) ≈ log(n) − 1/(2n) for large n
// Key use: 𝔼[log X] = ψ(k) when X ~ Gamma(k,1)

The digamma appears because we are taking expected values of logarithms of neighbor counts. If the number of points in a neighborhood follows a Poisson distribution with mean λ, then 𝔼[log(count)] = ψ(λ). This is why ψ(k) appears for the joint-space neighbor count and ψ(n_x + 1) appears for the marginal-space neighbor counts — we are computing expected log counts in different spaces.

Step-by-step derivation

①

Find the k-th neighbor in joint space

For each sample point (x_i, y_i), find its k-th nearest neighbor in the joint (X, Y) space using Chebyshev distance: d_∞((x,y),(x',y')) = max(|x−x'|, |y−y'|). Call this distance ε_i.

Why Chebyshev? It defines a hypercube neighborhood rather than a hypersphere, which simplifies the marginal counting in Step 2 — the projection of a Chebyshev ball onto a marginal axis is exactly the interval [x_i − ε_i/2, x_i + ε_i/2].

ε_i = d∞((x_i, y_i), k-th nearest neighbor)
// This radius defines a neighborhood of fixed cardinality (exactly k points)
// In high-density regions: ε_i is small
// In sparse regions: ε_i is large
②

Count neighbors in marginal spaces

Project the neighborhood radius ε_i onto each marginal axis separately and count how many points fall within that radius in each marginal.

n_x(i) = #{j ≠ i : |x_j − x_i| ≤ ε_i/2}
n_y(i) = #{j ≠ i : |y_j − y_i| ≤ ε_i/2}

// n_x(i) counts marginal X neighbors within the same radius
// If X and Y are independent: n_x ≈ n_y ≈ k on average (same density)
// If X and Y are dependent: n_x or n_y will be larger than k
// because marginal density is higher when you ignore the joint structure
③

The KSG formula and why it measures MI

KSG estimator (version 1)
Î(X;Y) = ψ(k) − ⟨ ψ(n_x + 1) + ψ(n_y + 1) ⟩ + ψ(N)

// ψ(k): expected log of joint-space neighbor count — fixed at k by construction
// ψ(n_x+1): expected log of marginal X neighbor count — varies per point
// ψ(n_y+1): expected log of marginal Y neighbor count — varies per point
// ψ(N): normalizing term for total sample size
// ⟨·⟩ denotes average over all N sample points

The structure mirrors the MI definition: log P(x,y) − log P(x) − log P(y). Each ψ term is the log density in the respective space, estimated via neighbor counts. When X and Y are independent, n_x and n_y are approximately k on average, so ψ(n_x+1) + ψ(n_y+1) ≈ 2ψ(k+1) ≈ 2ψ(k), and the formula returns approximately zero.

When X and Y are dependent, the joint space is more concentrated — the k-th neighbor is at ε_i — but the marginal spaces are more spread out, so n_x and n_y < k on average. The ψ(n_x+1) + ψ(n_y+1) terms decrease, the formula returns a positive value.

④

The Ross extension — discrete-continuous mixing

When one variable τ is discrete, its marginal distribution has point masses rather than a density. The standard KSG marginal neighbor count breaks down because all points with τ = τ_i are at distance 0 in the τ dimension.

Ross (2014) modification for mixed I(X; τ)
Î(X ; τ) = ψ(N) − ⟨ψ(Nτ)⟩ + ψ(k) − ⟨ψ(m_x + 1)⟩

// N_τ: number of samples with the same τ value as point i
// m_x: number of X-marginal neighbors within ε_i (continuous dimension only)
// ψ(N) − ψ(N_τ): contribution from the discrete variable
// ψ(k) − ψ(m_x+1): contribution from the continuous variable
// When τ has many values: N_τ is small, ψ(N)−ψ(N_τ) is large
// When τ is nearly constant: N_τ ≈ N, contribution ≈ 0

KSG Degradation Cases

When and why the KNN approach fails

Case 1 — Small N relative to dimension

KSG requires enough samples that each point's k-NN neighborhood contains a reliable local density estimate. When N is small relative to d, neighborhoods span large regions of the space — the k-th neighbor is far away — and the local density approximation breaks down. The estimator becomes high-variance rather than biased. Rule of thumb: N >> d for reliable estimates.

Case 2 — Manifold structure

KSG assumes samples are drawn from a distribution with support in the full ambient space. When data lies on a lower-dimensional manifold — typical for neural embeddings — Euclidean distance in the ambient space is a poor proxy for intrinsic geometry. Points far in ambient space may be close on the manifold. The solution: use a metric that respects manifold structure (cosine similarity, geodesic distance) rather than Euclidean distance.

Case 3 — Rare discrete values

The Ross extension requires enough samples per discrete value to estimate the continuous MI within each stratum. For rare edge types (few traversals of type τ = 'contrasts-with'), N_τ is small, and ψ(N_τ) has high variance. The per-stratum MI estimate is unreliable even when the total N is large.

Minimum samples per discrete stratum
Nτ >> dcontinuous · k
// For 768-dimensional embeddings and k=5: need N_τ >> 3840
// Rare edge types will not have this in early system phases

Case 4 — Latency at inference time

KSG requires O(N log N) neighbor search for each query. For offline Reliability(τ) estimation this is acceptable. For online ΔCoverage ranking at inference time, it is not — neighbor search over a large concept graph at query time is too slow. A fast proxy (cosine similarity between embeddings) is needed for the online case, with KSG used offline to calibrate that proxy.

For our setting KSG with Ross extension is the right estimator for offline Reliability(τ) computation from traversal logs, provided we have sufficient samples per edge type. Online ΔCoverage estimation requires a faster proxy — likely cosine similarity between query and candidate node embeddings, calibrated against KSG estimates on held-out data.

Neural MI Estimation (MINE)

Family 3 of 4 · Variational lower bound via learned critic network

Both histogram and KSG estimators require explicitly constructing neighborhoods or grids in the sample space. Neural estimators take a different route: they turn MI estimation into an optimization problem, training a neural network to distinguish joint samples from product-of-marginals samples.

MINE (Mutual Information Neural Estimation, Belghazi et al. 2018) uses the Donsker-Varadhan (DV) representation of KL divergence to express MI as the solution to a variational optimization:

Donsker-Varadhan representation
I(X;Y) = DKL(P(X,Y) ‖ P(X)⊗P(Y))
          = supT: Ω → ℝ { 𝔼P(X,Y)[T(x,y)] − log 𝔼P(X)⊗P(Y)[eT(x,y)] }

// T is any measurable function (the "critic")
// The supremum is achieved when T*(x,y) = log P(x,y)/P(x)P(y) + const
// We parameterize T as a neural network and optimize

The key advantage: no density estimation, no neighbor search, scales to very high dimensions, handles complex distributions that KSG cannot. The cost: requires training a neural network, which introduces variance and computational overhead.

MINE Mechanics

The variational formulation and training procedure
①

Parameterize the critic

Replace T with a neural network T_θ with parameters θ. The network takes a pair (x, y) as input and outputs a scalar.

Tθ : (x, y) → ℝ
// Architecture: MLP with ReLU activations, output is unbounded scalar
// The optimal T* separates joint from product-of-marginals samples
②

Sample joint and marginal pairs

From N samples {(x_i, y_i)}, construct two batches per training step:

Joint pairs: (x_i, y_i) ~ P(X,Y) — real co-occurring pairs
Marginal pairs: (x_i, y_j), i≠j ~ P(X)⊗P(Y) — shuffled, breaking dependence
// Shuffling Y independently of X creates samples from the product distribution
③

The MINE objective

MINE loss (maximize over θ)
Îθ(X;Y) = 𝔼joint[Tθ(x,y)] − log 𝔼marginal[eTθ(x,y)]

// First term: mean critic score on joint pairs — push up
// Second term: log mean exp on marginal pairs — push down
// Maximizing the gap trains T_θ to separate joint from marginal
// At optimum: T_θ*(x,y) ≈ log P(x,y)/P(x)P(y)
④

The variance problem and exponential moving average fix

The log 𝔼[e^T] term has high variance because exponentiation amplifies outliers. The gradient of this term is:

∇θ log 𝔼[eT] = 𝔼[eT · ∇θT] / 𝔼[eT]
// The numerator is weighted by e^T — large T values dominate
// A single outlier with large T_θ can make the gradient explode

MINE's fix: use an exponential moving average (EMA) of 𝔼[e^T] in the denominator rather than the batch estimate. This reduces variance at the cost of introducing a small bias proportional to the EMA decay rate.

EMA stabilization
Êt[eT] = α · Êt-1[eT] + (1−α) · batch estimate
// α close to 1: stable but biased (slow to track distribution shifts)
// α close to 0: unbiased but high variance

MINE Degradation Cases

When and why neural estimation fails

Case 1 — High variance (the fundamental problem)

MINE is asymptotically unbiased but has notoriously high variance. The log 𝔼[e^T] term amplifies any T value that's large — a single training step with a few high-T marginal samples can produce a wildly wrong gradient. This is not eliminated by EMA, only dampened.

Case 2 — Underestimation at high MI

When the true MI is large, the optimal T*(x,y) = log P(x,y)/P(x)P(y) takes on large values. The exponential in 𝔼[e^T] becomes dominated by extreme samples, making the denominator hard to estimate. MINE systematically underestimates MI in high-dependency regimes. Alternative estimators (NWJ/f-GAN, CPC) behave differently in this regime.

Case 3 — Requires sufficient training data

Unlike KSG which can work with N=30, MINE needs enough data to train the critic network reliably. The critic has its own capacity and generalization properties — with small N it memorizes the batch rather than learning the density ratio. Typical minimum: N in the hundreds to thousands depending on dimension.

Case 4 — Discrete-continuous mixing requires architecture choice

The critic T_θ(x, τ) must accept both continuous x and discrete τ. Standard approaches: embed τ as a learned embedding vector (treat it like a token), concatenate with x, pass through MLP. This works but the critic must learn the embedding jointly with the MI estimation, adding degrees of freedom and increasing training instability for rare τ values.

When neural estimation is appropriate Very high-dimensional continuous data where KSG is intractable. Complex non-Gaussian distributions. Large N available for training. Offline estimation where training cost is acceptable. For our Reliability(τ) use case with embedding-space queries and nodes: MINE is the natural choice if KSG neighbor search is too slow, but requires substantial traversal data per edge type.

Kernel & Copula Estimators

Family 4 of 4 · Smooth density estimation via bandwidth-controlled kernels

Kernel density estimation (KDE) takes a middle path between histograms (fixed grids) and KSG (pure geometry). Instead of assigning probability mass to bins, it places a smooth kernel function — typically Gaussian — at each sample point and sums these kernels to get a smooth density estimate.

Kernel density estimate
p̂_h(x) = (1/N) · Σi=1N Kh(x − x_i)
Kh(u) = (1/h) · K(u/h)    where K is the kernel (e.g. Gaussian)

// h is the bandwidth — controls smoothness
// Small h: spiky estimate, captures local structure, high variance
// Large h: smooth estimate, misses structure, high bias

The copula approach separates the problem differently: model the marginal distributions independently, then model only the dependence structure (the copula) between the probability-integral-transformed variables. This is particularly useful when marginals are complex but the dependence structure is tractable.

Kernel & Copula Mechanics

KDE-based MI estimation

①

Estimate the joint density p̂(x,y) using a multivariate kernel over the joint space. Estimate marginals p̂(x), p̂(y) using univariate kernels.

p̂(x,y) = (1/N) · Σi KH(x−x_i, y−y_i)
p̂(x) = (1/N) · Σi Kh_x(x−x_i)
p̂(y) = (1/N) · Σi Kh_y(y−y_i)
②

Plug into MI formula, integrating over the continuous space:

Î(X;Y) = ∫∫ p̂(x,y) · log [ p̂(x,y) / (p̂(x)·p̂(y)) ] dx dy
// The integral is approximated by Monte Carlo using the sample points

Copula approach

③

Transform each variable to uniform marginals using the probability integral transform: U = F_X(X), V = F_Y(Y). The copula C(u,v) = P(U≤u, V≤v) captures the pure dependence structure.

I(X;Y) = I(U;V) = ∫∫ c(u,v) · log c(u,v) du dv
// c(u,v) = ∂²C/∂u∂v is the copula density
// Working on [0,1]² removes marginal complexity
// Standard copulas (Gaussian, Clayton, Frank) can be fit parametrically
Bandwidth selection — the key decision The optimal bandwidth h* minimizes mean integrated squared error. For Gaussian kernels and Gaussian data: h* ∝ N^(−1/(d+4)). This is Silverman's rule. It assumes the data is roughly Gaussian — for multimodal or heavy-tailed distributions it oversmooths. Cross-validation is more robust but more expensive.

Kernel Degradation Cases

Case 1 — Bandwidth sensitivity

The MI estimate is sensitive to bandwidth choice. Undersmoothing (h too small) produces a spiky density that overfits the sample — the joint density appears more concentrated than it is, underestimating MI. Oversmoothing (h too large) blurs out the dependence structure, overestimating MI by failing to separate the joint from the product of marginals.

Case 2 — Dimensionality (same curse as histogram)

The optimal bandwidth scales as N^(−1/(d+4)). For d=768: h* ∝ N^(−1/772) ≈ 1 for any N less than astronomical. Kernel density estimation in high dimensions requires N exponential in d to achieve meaningful density estimates. Same fundamental issue as histograms, slightly more graceful.

Case 3 — Copula: marginal estimation error

The copula approach requires accurate estimation of F_X and F_Y to perform the probability integral transform. If the marginal CDF is estimated poorly (small N, or complex multimodal distribution), the transformed variables U and V are not uniform, corrupting the copula density estimate. The copula and marginal estimation errors do not cancel.

Case 4 — Mixed discrete-continuous

KDE for mixed variables requires mixed kernels — continuous kernels for continuous dimensions, discrete kernels (Aitchison-Aitken) for discrete dimensions. The bandwidth for the discrete dimension controls smoothing across discrete values, which is conceptually different from bandwidth in continuous space. The two bandwidth parameters interact and are hard to tune jointly.

When kernel estimation is appropriate Low-to-moderate dimension (d ≤ 5 or so). Sufficient N for bandwidth selection to work. When marginals are known or easily estimated (copula approach). When you need a smooth, differentiable density estimate rather than a point estimate of MI. For our setting: not competitive with KSG in most scenarios, but useful as a cross-check for 1D or 2D marginal MI estimates.

The meta-structure

All four families are attempting the same thing: estimate the ratio P(X,Y) / P(X)P(Y), whose expected log is I(X;Y). They differ in how they represent that ratio:

The common target
I(X;Y) = 𝔼P(X,Y) [ log P(X,Y) / (P(X)·P(Y)) ]

Histogram: represent densities as piecewise-constant functions on a grid
KSG: represent density ratios implicitly via neighbor count ratios
MINE: learn the log density ratio directly as a neural function
Kernel: represent densities as smooth kernel mixtures

// Each representation has different bias/variance tradeoffs
// Each fails when its representation assumptions are violated
// None is universally dominant — choice depends on data characteristics

Decision Guide

Which estimator for which setting
Q1

Are both variables discrete and low-cardinality (≤ 20 values)?

YES → Histogram. No density estimation needed. Bias is manageable. Fast and interpretable.

NO → Continue.

Q2

Is dimension d ≤ ~50 and do you have N >> d · k samples?

YES → KSG (with Ross extension if one variable is discrete). Consistent, well-characterized, no hyperparameter tuning beyond k. The default choice for moderate dimension offline estimation.

NO → Continue.

Q3

Is dimension very high (d > 50) and do you have large N for training?

YES → MINE or NWJ. Accept the variance cost. Use large batch size and EMA stabilization. Consider CPC (Contrastive Predictive Coding) as a lower-variance alternative that underestimates MI but has better training stability.

NO → Dimensionality-reduce first (PCA, UMAP to ≤20 dimensions), then KSG.

Q4

Do you need online MI estimation at inference time?

YES → Pre-train MINE offline. At inference use the critic Tθ(x,y) as a fast proxy — a single forward pass. Calibrate against KSG on held-out data to correct for MINE's variance and underestimation bias at high MI.

Alternatively: use cosine similarity between embeddings as a proxy, calibrated against offline KSG. O(1) at inference time.

PKIS Application

Mapping the three MI estimation problems to estimator choices

Reliability(τ) — offline edge type calibration

Estimate E[I(q; c2 | c1, τ) − I(q; c2)] from traversal logs. Query embedding q is continuous high-d. Edge type τ is discrete low-cardinality. The key constraint is Nτ per edge type.

Recommended: KSG + Ross extension Offline computation — latency is not a constraint. Pool all traversals of each τ value. If Nτ < 200 per stratum, estimates will be noisy — flag those edge types as insufficiently calibrated and fall back to LLM judgment (Mode 2 in the hardening framework). For high-frequency edge types: KSG is reliable and tractable.

ΔCoverage(q, c | S) — online traversal ranking

Rank frontier nodes at inference time by estimated marginal MI gain. Latency-constrained.

Recommended: Cosine similarity proxy, calibrated offline Cosine similarity between query embedding and candidate node embedding. O(1) per candidate. Calibrate the mapping from cosine similarity to estimated MI gain using offline KSG estimates on held-out (query, node) pairs. Fit isotonic regression or sigmoid mapping.
